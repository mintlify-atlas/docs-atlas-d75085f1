## Overview

The `AaveBorrower` contract is a protocol-specific borrower implementation that enables the FlashLoanRouter to request flash loans from Aave V3 pools. It acts as an adapter between the router's generic interface and Aave's specific flash loan mechanism.

### Key Features

- **Aave V3 Integration**: Implements Aave's flash loan receiver interface
- **Router Coordination**: Seamlessly integrates with FlashLoanRouter
- **Zero Debt Mode**: Requests flash loans without opening debt positions
- **Generic Borrower Base**: Inherits common borrower functionality

### How It Works

1. Router calls `flashLoanAndCallBack()` on this contract
2. Contract requests flash loan from Aave pool using `flashLoan()`
3. Aave transfers funds and calls `executeOperation()`
4. Contract processes loan and calls back the router
5. Flash loan is repaid during settlement execution

## Internal Implementation

### triggerFlashLoan

```solidity
function triggerFlashLoan(
    address lender,
    IERC20 token,
    uint256 amount,
    bytes calldata callBackData
) internal override
```

Internal function that implements Aave-specific flash loan request logic.

**Implementation Details:**

```solidity
IAaveFlashLoanReceiver receiverAddress = this;
address[] memory assets = new address[](1);
assets[0] = address(token);

uint256[] memory amounts = new uint256[](1);
amounts[0] = amount;

uint256[] memory interestRateModes = new uint256[](1);
interestRateModes[0] = 0; // No debt position

address onBehalfOf = address(this);
bytes calldata params = callBackData;
uint16 referralCode = 0; // No referral

IAavePool(lender).flashLoan(
    address(receiverAddress),
    assets,
    amounts,
    interestRateModes,
    onBehalfOf,
    params,
    referralCode
);
```

**Key Parameters:**
- `interestRateModes[0] = 0`: Specifies no debt position (must repay in same transaction)
- `referralCode = 0`: Referral system currently inactive on Aave
- `onBehalfOf = address(this)`: Loan taken on behalf of this contract

## Aave Flash Loan Flow

1. **Request**: Router calls `borrower.flashLoanAndCallBack()`
2. **Trigger**: Borrower calls `aavePool.flashLoan()`
3. **Transfer**: Aave transfers tokens to borrower contract
4. **Callback**: Aave calls `borrower.executeOperation()`
5. **Notify**: Borrower calls `router.borrowerCallBack()`
6. **Settlement**: Router executes CoW Protocol settlement
7. **Approval**: Settlement calls `borrower.approve()` for Aave pool
8. **Repayment**: Aave retrieves funds via `transferFrom()` from borrower
9. **Verification**: Aave verifies repayment amount (principal + fee)

## Aave V3 Integration Details

### Flash Loan Parameters

- **Interest Rate Mode**: Set to `0` (no debt position)
  - Mode `0`: Flash loan must be repaid in same transaction
  - Mode `1`: Stable debt (not used)
  - Mode `2`: Variable debt (not used)

- **Referral Code**: Set to `0` (inactive)
  - Aave's referral program is currently not active

- **Premium (Fee)**: Variable per asset
  - Typically 0.05% - 0.09% of borrowed amount
  - Check Aave documentation for current rates
  - Must be included in repayment amount

### Repayment Mechanism

Aave uses the "pull" model:
1. Borrowed funds transferred to borrower
2. After callback completes, Aave calls `transferFrom()` on borrower
3. Borrower must have approved Aave pool for `amount + premium`
4. Settlement interactions must set up this approval

## Security Considerations

### Access Control
- `flashLoanAndCallBack()`: Only callable by registered router
- `approve()`: Only callable by settlement contract
- `executeOperation()`: Only callable by Aave pool (validated by router flow)

### Flash Loan Safety
- **No Debt Positions**: Interest rate mode set to 0 prevents debt accumulation
- **Same Transaction**: Loan must be repaid before transaction completes
- **Approval Control**: Settlement controls all token approvals

### Data Integrity
- Callback data passed through without modification
- Router validates data hash to ensure integrity
- No state changes that could affect router logic

### Aave-Specific Risks

<Warning>
  Ensure the lender address is a legitimate Aave V3 Pool contract. Using an incorrect address could result in loss of funds.
</Warning>

- **Premium Calculation**: Settlement must account for Aave's flash loan fee
- **Asset Availability**: Requested amount must be available in Aave pool
- **Approval Timing**: Approval for repayment must occur during settlement
