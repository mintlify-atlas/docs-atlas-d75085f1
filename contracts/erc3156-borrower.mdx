## Overview

The `ERC3156Borrower` contract is a standardized borrower implementation that enables the FlashLoanRouter to request flash loans from any lender that implements the ERC-3156 flash loan standard. This provides a universal interface for flash loan providers that follow the standard.

### Key Features

- **Standard Compliance**: Implements ERC-3156 flash borrower interface
- **Universal Compatibility**: Works with any ERC-3156 compliant lender
- **Simple Integration**: Standardized callback mechanism
- **Success Validation**: Verifies flash loan success before proceeding

### Supported Lenders

Any flash loan provider implementing ERC-3156, including:
- Maker's flash mint module
- Yield Protocol
- Euler Finance
- Custom ERC-3156 implementations

## ERC-3156 Standard

ERC-3156 defines a standardized interface for flash loans:
- **Lender Interface**: `IERC3156FlashLender`
  - `flashLoan()`: Initiates a flash loan
  - `maxFlashLoan()`: Returns maximum borrowable amount
  - `flashFee()`: Returns fee for a given amount

- **Borrower Interface**: `IERC3156FlashBorrower`
  - `onFlashLoan()`: Callback after receiving funds
  - Must return success constant

## Internal Implementation

### triggerFlashLoan

```solidity
function triggerFlashLoan(
    address lender,
    IERC20 token,
    uint256 amount,
    bytes calldata callBackData
) internal override
```

Internal function that implements ERC-3156 flash loan request logic.

**Implementation:**

```solidity
bool success = IERC3156FlashLender(lender).flashLoan(
    this,                    // receiver
    address(token),          // token to borrow
    amount,                  // amount to borrow
    callBackData             // data to pass through
);
require(success, "Flash loan was unsuccessful");
```

**Parameters to ERC-3156 flashLoan:**
- `receiver`: This borrower contract (will receive callback)
- `token`: Address of token to borrow
- `amount`: Amount to borrow
- `data`: Arbitrary data passed to callback

## Usage Example

```solidity
// Deploy borrower (one-time setup)
FlashLoanRouter router = FlashLoanRouter(routerAddress);
ERC3156Borrower borrower = new ERC3156Borrower(router);

// Prepare flash loan through router
address erc3156Lender = 0x...; // Any ERC-3156 compliant lender
IERC20 dai = IERC20(0x...);
uint256 borrowAmount = 100_000e18; // 100,000 DAI

// Query flash loan fee from lender (optional)
IERC3156FlashLender lender = IERC3156FlashLender(erc3156Lender);
uint256 fee = lender.flashFee(address(dai), borrowAmount);
uint256 totalRepayment = borrowAmount + fee;

// Create loan request
Loan.Data memory loan = Loan.Data({
    amount: borrowAmount,
    borrower: IBorrower(address(borrower)),
    lender: erc3156Lender,
    token: dai
});

// During settlement, approve lender for repayment
// borrower.approve(dai, erc3156Lender, totalRepayment);
```

## ERC-3156 Flash Loan Flow

1. **Request**: Router calls `borrower.flashLoanAndCallBack()`
2. **Trigger**: Borrower calls `lender.flashLoan()`
3. **Transfer**: Lender transfers tokens to borrower contract
4. **Callback**: Lender calls `borrower.onFlashLoan()`
5. **Notify**: Borrower calls `router.borrowerCallBack()`
6. **Settlement**: Router executes CoW Protocol settlement
7. **Approval**: Settlement calls `borrower.approve()` for lender
8. **Repayment**: Lender retrieves funds via `transferFrom()` from borrower
9. **Return**: Borrower returns success constant to lender
10. **Verify**: Lender verifies repayment and returns success to borrower

## ERC-3156 Integration Details

### Flash Loan Parameters

ERC-3156 lenders provide standardized query functions:

```solidity
IERC3156FlashLender lender = IERC3156FlashLender(lenderAddress);

// Check maximum borrowable amount
uint256 maxAmount = lender.maxFlashLoan(address(token));

// Query flash loan fee
uint256 fee = lender.flashFee(address(token), amount);

// Total repayment required
uint256 totalRepayment = amount + fee;
```

### Repayment Mechanism

ERC-3156 uses the "pull" model similar to Aave:
1. Borrower receives funds before callback
2. Callback executes (router notified, settlement executed)
3. Lender calls `transferFrom()` to retrieve funds
4. Borrower must have approved lender for `amount + fee`
5. Callback must return success constant

### Success Validation

The contract validates two success indicators:

1. **Callback Return**: Must return `keccak256("ERC3156FlashBorrower.onFlashLoan")`
2. **Flash Loan Result**: `flashLoan()` must return `true`

If either fails, the transaction reverts.

## Security Considerations

### Access Control
- `flashLoanAndCallBack()`: Only callable by registered router
- `approve()`: Only callable by settlement contract
- `onFlashLoan()`: Only callable by lender (validated by router flow)

### Standard Compliance
- **Success Constant**: Must return exact ERC-3156 success value
- **Callback Signature**: Must match ERC-3156 specification
- **Repayment Timing**: Approval must occur before callback returns

### Data Integrity
- Callback data passed through without modification
- Router validates data hash to ensure integrity
- No state changes that could affect router logic

### ERC-3156 Specific Risks

<Warning>
  Verify the lender implements ERC-3156 correctly before using it. Non-compliant implementations may cause unexpected behavior or loss of funds.
</Warning>

- **Lender Validation**: Ensure lender address is a legitimate ERC-3156 implementation
- **Fee Calculation**: Query `flashFee()` to determine exact repayment amount
- **Max Amount**: Check `maxFlashLoan()` to ensure requested amount is available
- **Approval Timing**: Settlement must approve lender before callback completes

### Testing ERC-3156 Compliance

```solidity
// Verify lender implements ERC-3156
try IERC3156FlashLender(lender).maxFlashLoan(token) returns (uint256) {
    // Lender supports the interface
} catch {
    // Lender may not be ERC-3156 compliant
}

// Verify fee is reasonable
uint256 fee = lender.flashFee(token, amount);
require(fee < amount, "Fee exceeds borrowed amount");
```
